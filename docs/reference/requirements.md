# 要件

## 外部的要件

* ウェブアプリケーションであること。
* ブラウザローカルで動作すること。サーバサイドで処理しないこと。
* 初回アクセス時は Start 操作後に演奏が始まる。
* 以後のアクセスや同一セッション内では、ブラウザが許す範囲で可能なら自動再開する。
* 演奏中はビジュアライザで譜面を可視化する。
* 譜面を遡ることができること。MVP では直近 10 分程度を目安に、メモリ上のリングバッファで保持する。
* パート数：4（ソプラノ、アルト、テナー、バス）を基本とする。将来的にはピアノ譜面相当を生成できることを目標とする。

## 技術的目標

* Web Worker / SharedWorker / AudioWorklet を使ってマルチCPUで処理する。
  * Service Worker は計算並列化ではなく、オフライン対応やキャッシュ用途として扱う。
* WebAudio によるシンセサイザをもっていること。演奏はシンセサイザで行うこと。
* フーガ生成のアルゴリズムは、以下をもとにして構築すること：
  * これまでに演奏してきた譜面（コンテキスト）から、最適な続きを計算する状態マシン
  * 初期値は、数小節程度の主題を生成するアルゴリズムで生成した主題を使う。
  * 音楽理論は Fux の教本の対位法を基本にする。
  * フーガの展開は J.S. Bach のフーガの音楽理論的研究をもとにする。
